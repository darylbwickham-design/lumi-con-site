Why this setup

A simple landing page, a browser installer, and a clear version history.

Clear install path

Users land on a proper device page first, then go straight to the installer.

Recent as default

The recent folder always points to the build you want most users to install.

Older versions available

The previous three versions stay available from the dropdown for rollback, testing, or recovery.

How it works

Static hosting, browser flashing, and a simple release layout.

Host the site

Upload this folder to HTTPS hosting. Static hosting is enough for the site, assets, manifests, and firmware bins.

Keep recent current

Drop the recommended build into firmware/recent/ so the default option always points where you want.

Archive versions

Keep version-specific manifests and bins inside firmware/versions/ so users can pick older releases from the dropdown.

Flash from browser

Users choose their device, pick a version, connect by USB, and install through desktop Chrome or Edge.

Tip: use a proper data USB cable, serve the site over HTTPS, and keep your firmware filenames consistent.
